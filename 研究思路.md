# ==================== 一、研究背景与动机

我的研究目标不是传统意义上的分类攻击，而是“图像保护”。

具体场景是：

- 我有一些待保护的图像。
- 攻击者可能拿这些图像去喂给 image-to-video（i2v）模型，恶意生成正常视频。
- 我希望对原图施加对抗性扰动，使图像在人眼上尽量保持自然，但一旦被用于 i2v，就难以生成正常、稳定、可用的视频。

因此，这不是单纯的“把图像分类到某个目标类别”的问题，而是一个更偏向：

- 图像保护
- 下游生成失稳
- 扰动可迁移
- 兼顾隐蔽性

的研究问题。

# ==================== 二、为什么选择 UnivIntruder 作为起点

我之所以基于 UnivIntruder 开始，是因为它已经验证了一条对我很有启发的思路：

- 在优化阶段，不直接依赖目标模型本身
- 而是用通用 surrogate（例如 CLIP / SigLIP / ImageBind 一类模型）来学习具有迁移性的扰动
- 其核心机制是：用图像方向和文本方向对齐，构造 targeted、transferable 的 universal perturbation

这与我的研究方向有重合点：我也希望在不知道最终目标模型细节的情况下，借助通用视觉语义模型学习可迁移的保护扰动。

但我要强调：
**UnivIntruder 只是方法原型，不是最终方案。**
我不会把它原封不动当成最终系统，而是要在它的基础上逐步改造成面向 i2v 保护的框架。

# ==================== 三、我对当前项目的核心判断

这是后续开发最重要的共识，请严格按这个理解项目。

1. 当前原始 UnivIntruder 更接近：

- 通用 UAP
- 目标语义劫持
- 面向分类 / 检索 / VLM 的 transferable attack

而我的最终目标更接近：

- 面向单张图像的保护
- 扰动尽量不显眼
- 下游视频生成过程失稳或异常
- 迁移性仍重要，但不是唯一指标

1. 我目前已经观察到：

- 原仓库在一些目标概念下，容易出现肉眼可见的语义伪影，例如 hen 类伪影
- 这说明原方法“高攻击性”与“高隐蔽性”之间存在明显张力

1. 我当前的判断是：

- 可见伪影不只是训练没跑好，也和原始方法本身的归纳偏置有关
- 关键原因包括：
  - 共享 UAP 形式
  - 较强的随机可微增强
  - loss 中没有显式的隐蔽性约束
- 所以不能只靠“继续复现原论文”解决问题，必须逐步改造目标函数与优化形式

# ==================== 四、项目的总目标

请始终以以下目标作为后续执行方向：

总目标：
**在尽量不破坏原图视觉质量的前提下，构造对 i2v 流程具有迁移性的保护扰动。**

这意味着后续代码和实验都应该围绕以下三件事展开：

1. 迁移性

- 扰动不能只对单个目标模型有效
- 要尽可能通过通用 surrogate 学到跨模型、跨流程可迁移的信号

1. 隐蔽性

- 保护图像不能明显出现目标概念伪影
- 人眼视觉质量是关键评价维度，不是附属指标

1. 下游失稳性

- 最终不是追求“被分类成某个 target class”
- 而是追求“图像进入 i2v 后，视频生成结果出现明显异常、失稳、不可用或不可信”

# ==================== 五、当前的分阶段策略

请把后续执行理解为分阶段推进，而不是一次性大改。

------

## 阶段 1：理解并校准原始 UnivIntruder

目标：

- 理解原始代码结构与关键模块
- 保持原始 UAP 主线可运行
- 找到会明显影响图像可见性的关键实现因素

重点：

- 不随意破坏原 main.py / evaluate.py 主链路
- 能区分“原方法问题”和“实现设置问题”
- 原始仓库应始终保持可用

------

## 阶段 2：从 UAP 扩展到 per-image protector

这是当前正在进行的阶段。

目标：

- 在不破坏原始 UAP 路线的前提下，新增 per-image 分支
- 将“共享 trigger”扩展为“每张图单独优化一个 delta”
- 仍复用现有 surrogate 思路（例如 CLIP directional loss）
- 这是从“攻击任意输入”转向“保护某张图像”的关键过渡

当前阶段边界：

- 当前阶段不要引入 LPIPS / TV / low-frequency / instability loss
- 只做结构扩展：per-image 独立优化
- 原始 UAP 路线必须保留，不能被覆盖或破坏

------

## 阶段 3：显式加入隐蔽性约束

后续计划，不是当前优先任务。

目标：

- 在 per-image 基础上，加入感知质量约束
- 重点缓解可见语义伪影
- 倾向于加入 LPIPS / TV / low-frequency penalty / 频域约束 等

------

## 阶段 4：加入面向 i2v 的失稳性目标

后续计划，不是当前优先任务。

目标：

- 不再把“targeted concept hijack”当唯一优化方向
- 而是加入多视图不稳定性 / 语义一致性破坏等 surrogate 目标
- 更直接地服务于 i2v 保护任务
